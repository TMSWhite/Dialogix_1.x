Triceps

Abstract

Many people hope to use the Internet as a forum in which they can express their opinions via on-line polls, explore their interests via on-line education and testing, and seek medical advice tailored to their individual circumstances. Many businesses are striving to provide these services, but are often hindered by economic, technological, political, and sociological challenges. These hurdles have also made if difficult for researchers to identify and evaluate new and better ways to provide these services. The goal of the Triceps project was to help researchers lower these hurdles and speed the development of improved services.

Communication barriers contribute to many of these hurdles, which is not surprising given how many different groups and disciplines are necessary to design, build, test, and market improved services. The informatics fields of knowledge acquisition, representation, and transfer address the challenge of getting and expert to codify their knowledge in a fashion that a programmer can implement.

      • criteria to be exempted (Category I)
        • established or commoly accepted eductation settings involving normal education practices
          • assessing instructional strategies
          • effectiveness of or comparison among instructional techniques, and curricula
        • using educational tests (cognitive, diagnostic, aptitude, achievement), surveys, interviews, or observation of public behavior when the subject cannot be identified [what constitutes unidentifiable?]
          • not exempt when:
            • applies to children
            • subjects can be identified directly through identifiers linked to the subjects
            • any disclosure of responses outside of research could reasonably place subject at risk of criminal or civil liability, or be damaging to subject's financial standing, employability or reputation
            • subjects are involved in politics
            • federal statutes require that confidentiality of personally identifiable info be maintained in perpetuity
        • if no way to identify subjects, then can use publically available existing data, documents, records, pathological and diagnostic specimens
      • criteria to be expedited (Category II)
        • interview and interactive surveys where subjects are identified.
          • full review required if involves topics identified by Federal Register as "sensitive"
            • illegal conduct, drug use, sexual behavior, alcohol use
        • confidential, individually non-identifiable use of existing data, documents, or medical records
        • individual and group behavior (perception, cognition, game theory, test development) where subject's behaviors are not manipulated and research will not stress the subjects
        • collection of data through non-invasive, standard clinical procedures already being performed for diagnostic or treatment purposes

Security Notes (from SANS - How to Autid Web-Based Applications)

  • Top ten most Critical Security Risks
    • http://www.sans.org/topten.htm
      • BIND - DNS - named
      • CGI and ColdFusion Sample Programs
      • RPC - Tooltalk, Calendar Manager, rpc.statd
      • RDS (remote data services under IIS)
      • Sendmail and MIME buffer overflows
      • sadmind and mountd buffer overflows (Solaris & NFS)
      • global file sharing: NetBIOS, Windows NT/2000, AppleShare/IP, NFS exports
      • weak passwords
      • IMAP and POP buffer overflow
      • SNMP default passwords
      • Microsoft Internet Explorer / Outlook - all scripting and ActiveX; .ISM.dll (.htr, .tsm, .idc)
  • Foundational Problems
    • indexable directories
    • default files
      • showcode.asp
      • webdist.cgi
      • php.cgi
      • /scripts/tools/newdsn.exe
      • perl.exe - if available in path, can pass arbitrary commands
      • phf phone book script in NCSA and Apache
  • Web Server Output
    • Server: field - says which server is being used - can be exploited
    • telnet - can telnet to the web-server port (80) & use GET command to determine the type of server
    • HTML & Javascript
      • developer names (social engineering)
      • disabled functionality
      • third party tools in use (with known vulnerabilities - like Coldfusion)
    • Mirroring the site
      • WebReaper (http://www.otway.com/webreaper)
      • Wget (http://freshmeat.net/search.php3?query=wget)
    • Robots
      • /robots.text
        • User-agent: * # applys to all browsers
        • Disallow: / # disallows all
      • <meta name="robots" content="none"/>
    • Accidental error messages
      • ones that show differences re username/password - exploitable via brute force (e.g. MCI Worldcom)
      • Coldfusion default message shows SQL command
  • Sign-on security
    • types
      • user name harvesting
      • brute force password guessing
      • automated account lock-outs
      • harvesting & reaping
      • password harvesting
      • resource exhaustion
    • solution
      • add dynamic temp key for each step in a session or for each sign-on step
      • check the HTTP Referer field
  • Session Issues
    • don't expose the session key - since equivalent to username + password
    • cookies safer then hidden IFF marked secure and non-persistent
    • sessionIDs are fine as long as they are random, long, unrelated to user info, perishable, securely sent
      • might also include crypt of source IP so that not spoofed, or keyed on domain (in case of poor ISP connection)
      • http://www.apl.jhu.edu/~hall/java/Servlet-Tutorial/Servlet-Tutorial-Session-Tracking.html
    • IP Hopping - enables Session Cloning
    • Concurrency
      • don't allow two sign-ons from same username
      • auto-logoff 1st account if try to login again (assume a disconnect) ONLY if fully authenticate second session
  • Transaction Issues
    • unexpected input (XForms does not deal with this, since purely client side JavaScript)
    • use POST instead of GET (since GET is logged in history and web-server logs)
    • Referer field - will send sensitive GET string info
      • http://www.gemal.dk/browserspy/document.html
    • Improper logic sequence (transactions out of order)
  • Third Party Products
    • Web-based Certificate Authorities (CA)
      • Microsoft Certificate Server
        • http://www.foobar.firm/CertAdm/wcaForm.asp - reveals user name, company name, email address, location for all users
      • Entrust WebCA and Entrust/Web Connector
        • https://bigbank.firm/cgi-cli/webcli.exe?type=retrievecert
    • Other
      • WebTrends Enterprise Reporting Server v. 1.5
      • Cold Fusion Application Server v 2.0, 3.0, 3.1, 4.0
    • Remove <meta name="generator"> fields - gives it away
  • Testing
    • HTTP proxy sniffer (e.g. HTTP ProxySpy @ http://home.comset.net/povarov)
    • test buffer overflows & invalid inputs for all variables
    • look for overly informative error messages
